## Context

《影梦》项目已完成世界观设定、美术风格确定，但核心玩法机制尚未确定。团队经验有限，需要一个简单但有效的方案来推动开发。

**约束条件：**
- Godot 4.x引擎
- 千禧年梦核风美术风格
- 已决定不做战斗
- 团队经验有限，适合从简入手

## Goals / Non-Goals

**Goals:**
- 定义清晰的核心玩法循环
- 设计适合新手团队实现的机制
- 整合睡眠阶段系统
- 为Agent A和Agent C提供明确的需求方向

**Non-Goals:**
- 不实现战斗系统
- 不做复杂的RPG成长机制
- 不追求高难度玩法（第一版侧重体验）
- 不实现所有特殊睡眠状态（仅实现基础四个阶段）

## Decisions

### 1. 核心体验类型
**决定：** 探索解谜导向（无战斗）

**理由：**
- 与剧情和氛围契合度高
- 程序复杂度低，适合新手团队
- 美术资源需求适中（不需要战斗动画）
- 与《小小梦魇》《Gris》等参考游戏风格一致

**备选方案：**
- 动作战斗导向：复杂度太高，不适合当前团队
- 叙事体验导向：游戏性可能太弱
- 混合机制：范围太大，不适合第一版

### 2. 核心玩法循环
**决定：** 探索→收集记忆碎片→解谜/躲避→推进剧情

**理由：**
- 简单明了，容易实现
- 能充分利用睡眠阶段系统
- 与叙事结合紧密

### 3. 玩家操作设计
**决定：** 极简四键操作

| 操作 | 功能 |
|------|------|
| 方向键/WASD | 移动 |
| E/空格 | 互动 |
| Shift | 快速行走 |
| Ctrl | 躲藏 |

**理由：**
- 新手友好
- 容易实现和测试
- 符合冒险解谜游戏的操作习惯

### 4. 睡眠阶段实现
**决定：** 简化版四阶段，去掉复杂特殊状态

| 阶段 | 核心玩法 |
|------|----------|
| 清醒期 | 选择关卡（UI界面） |
| 浅睡期 | 探索+简单解谜+躲避巡逻敌人 |
| 深睡期 | 紧张躲避+恐惧值管理 |
| REM期 | 情感共鸣+高潮解谜 |

**理由：**
- 保留核心设定，但降低实现难度
- 特殊状态（梦中梦、清醒梦、鬼压床）可以后续版本再加
- 四个阶段足够支撑第一版游戏体验

### 5. 敌人设计
**决定：** 非战斗型敌人，以躲避为主

| 类型 | 行为 |
|------|------|
| 巡逻型 | 固定路线来回移动 |
| 追踪型 | 发现玩家后追击 |
| 潜伏型 | 静止不动，触发后惊吓 |

**理由：**
- 不需要战斗系统实现
- 符合情绪投射设定（敌人是情绪的具象化，不是消灭对象）
- 与《小小梦魇》风格一致

### 6. 技术实现方案
**决定：** Godot 4.2稳定版，2D引擎+分层视差

**理由：**
- Godot 4.2是当前最稳定的版本
- 2D引擎适合像素风格游戏
- 分层视差可以实现伪3D效果，符合美术风格
- 团队可以快速上手

## Risks / Trade-offs

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| 玩法可能太简单 | 玩家体验不足 | 设计多层次谜题，提供简单和困难两种模式 |
| 恐惧值系统可能不够明显 | 情绪体验不足 | 加强视觉和音效反馈 |
| 四个阶段可能节奏不对 | 体验不流畅 | 先做原型测试，调整时间和难度 |
| 去掉战斗可能导致某些设定浪费 | 医学元素应用受限 | 将医学元素转化为谜题和异常状态设计 |
