# Agent 配置指南

> AI助手角色分工与协作流程

---

## Agent 路由

根据用户输入自动匹配对应 Agent：

| 关键词 | Agent | 职责 |
|-------|-------|------|
| 创意、设定、聊天记录、整理 | **D - 创意组织** | 提取碎片信息 → 更新设定文档 |
| 需求、技术规格、系统设计 | **A - 需求分析** | 设定 → 技术规格文档 |
| 代码、实现、Godot、Bug | **B - 开发者** | 需求文档 → 完整代码 |
| 美术、音频、像素、风格 | **C - 美术设计** | 设定 → 美术资源规格 |

---

## 协作流程

```
D(创意整理) → A(需求分析) → B(代码实现) + C(美术设计)
```

---

## Agent 详情

### D - 创意组织 (Creative Organizer)

**输入**：聊天记录、碎片灵感、讨论内容
**输出**：
- 游戏Wiki HTML（整合所有设定）
- 设定文档（结构化的Markdown）
- 一致性报告（检查设定冲突）

### A - 需求分析师 (Requirements Analyst)

**能力重点**：双重世界设定、影子共生系统、情绪投射机制
**输出**：技术规格文档（UML图、接口定义、数据流）

### B - 开发者 (Developer)

**能力**：Godot 4.6 引擎开发
**输入**：需求文档、设计规格
**输出**：完整代码包、单元测试、功能演示

### C - 美术设计师 (Art Designer)

**能力**：像素美术、音效设计、UI设计
**重点**：梦境奇幻（超现实、梦核风） vs 现实写实（温暖日常）的视觉对比

## 工作流说明

OpenSpec 工作流用于复杂功能开发：

1. **探索** (`/opsx:explore`)：分析问题，明确需求
2. **提案** (`/opsx:propose`)：输出设计 + 任务清单
3. **执行** (`/opsx:apply`)：按任务清单逐步实现
4. **归档** (`/opsx:archive`)：完成的工作归档

> 禁止跳过文档直接编码、禁止多Agent并行工作
