## 🎮 瓶颈 #1：核心玩法机制（Critical）

### 问题描述

> **核心矛盾**：我们有了完整的剧情和世界观，但"这个游戏要怎么玩？"

目前的设定已经非常丰富：
- ✅ 影子共生系统（依附/游离/消逝）
- ✅ 情绪投射机制（恐惧→怪物、孤独→迷宫）
- ✅ 睡眠阶段系统（清醒期→浅睡期→深睡期→REM期）
- ✅ 医学元素设计（疾病→异常状态、病原→怪物、心理→技能）

**但核心玩法循环仍未确定**，这直接导致：
- Agent A 无法确定程序需求文档的核心系统规格
- Agent C 无法确定美术资源的动作需求（是否需要战斗动画？）
- 无法确定音效需求（战斗音效？环境音效？）
- 整个第二阶段 Agent 协调工作无法推进

---

### 决策结果

经过讨论，我们决定采用**探索解谜导向**的核心玩法，具体方案如下：

#### 1.1 核心体验类型（已确定）

- ✅ **选项B：探索解谜导向**
  - 环境解谜，记忆收集，剧情推进
  - 参考：Gris、Journey、Inside
  - 优势：与叙事和氛围契合度高
  - 挑战：需要设计有深度的解谜机制

**核心循环**：
```
进入梦境 → 探索场景 → 收集记忆 → 解谜/躲避 → 推进剧情
```

#### 1.2 战斗系统（已确定）

- ✅ **无战斗设计**：保留追逐、躲避等紧张元素，重点放在剧情和情感体验上
- **敌人处理**：通过躲避、安抚、解谜等方式与敌人互动，而非战斗

#### 1.3 睡眠阶段与玩法结合（已确定）

| 睡眠阶段 | 玩法表现 |
|---------|----------|
| 清醒期 | 选择梦境区域，查看记忆碎片，设置目标 |
| 浅睡期 | 探索 + 简单解谜 + 躲避巡逻型敌人 |
| 深睡期 | 紧张躲避 + 恐惧管理 + 资源收集 |
| REM期 | 高潮解谜 + 情感共鸣 + 剧情解锁 |

---

### 具体玩法设计

#### 第一阶段：清醒期（5分钟）
- **玩家做什么**：选择今天要去的梦境区域（类似选关卡）
- **简单机制**：看一段之前收集的记忆碎片，选择今天的目标
- **视觉表现**：躺在床上，视角逐渐模糊，出现几个梦境入口

#### 第二阶段：浅睡期（10分钟）
- **玩家做什么**：探索 + 解谜 + 躲避巡逻型敌人
- **简单机制**：
  - 场景会"微微晃动"（表现梦境的不稳定）
  - 找到3个记忆锚点来稳定场景
  - 避开巡逻的情绪怪物
- **谜题类型**：简单的环境解谜（推箱子、找开关）

#### 第三阶段：深睡期（10分钟）
- **玩家做什么**：更紧张的躲避 + 资源管理
- **简单机制**：
  - 恐惧值缓慢上升，高了画面会模糊
  - 找到安全点可以降低恐惧
  - 收集记忆碎片（但数量更少）
- **敌人**：更聪明的追踪型敌人

#### 第四阶段：REM期（15分钟）
- **玩家做什么**：高潮解谜 + 情感释放
- **简单机制**：
  - 不是战斗，而是**情感共鸣**
  - 通过与场景互动来"安抚"情绪怪物
  - 解开核心谜题，解锁剧情片段
- **视觉表现**：更梦幻、更夸张的场景

#### 玩家操作设计（极简版）

| 操作 | 功能 | 说明 |
|------|------|------|
| 方向键/WASD | 移动 | 影子移动 |
| E/空格 | 互动 | 捡东西、拉开关、安抚敌人 |
| Shift | 快速行走 | 消耗小，但可能吸引敌人 |
| Ctrl | 躲藏 | 躲在物体后面，不被发现 |

### 决策辅助信息

#### 参考游戏深度分析

| 游戏 | 核心机制 | 与《影梦》的契合点 | 可借鉴之处 |
|-----|---------|------------------|-----------|
| **Hollow Knight** | 类银河战士恶魔城+精准战斗 | 梦境主题、情绪化的敌人设计 | 地图探索、战斗节奏 |
| **Hades** | 动作Roguelike+叙事 | 神话背景、角色关系 | 战斗build、叙事整合 |
| **Gris** | 平台解谜+情感叙事 | 情感具象化、美术风格 | 无战斗的叙事方式 |
| **以撒的结合** | 射击Roguelike | 宗教隐喻、道具组合 | 随机性、重玩价值 |
| **Celeste** | 平台跳跃+心理叙事 | 心理主题、章节结构 | 难度曲线、叙事节奏 |

#### 团队资源考量

| 因素 | 战斗导向 | 解谜导向 | 叙事导向 |
|-----|---------|---------|---------|
| 程序复杂度 | 高 | 中 | 低 |
| 美术资源量 | 高（动作、特效） | 中（环境、动画） | 低（场景、过场） |
| 设计难度 | 高（平衡、build） | 中（谜题设计） | 低（线性叙事） |
| 与剧情契合 | 中 | 高 | 极高 |
| 市场接受度 | 高 | 中 | 低 |

## 📖 标准化工作流程

### 瓶颈处理流程

当本文档中的瓶颈得到解决时，按以下步骤处理：

```
步骤1: 标记状态
├── 在优先级总览表中将该项标记为"✅ 已确定"或"✅ 已解决"
└── 添加决策结果简述

步骤2: 样式处理
├── 在文档主体中，为该瓶颈标题添加删除线（~~标题~~）
├── 在表格中添加删除线标记
└── 使用灰色/淡化效果降低视觉突出度

步骤3: 信息归档
├── 将已解决的瓶颈完整内容复制到"已解决的瓶颈（归档区）"
├── 保留关键决策信息（日期、结果、负责人）
├── 添加关联文档链接
└── 可选：添加决策背景简述

步骤4: 文档清理
├── 从优先级总览表中移除或降级该项
├── 更新待办清单，移除相关任务
└── 更新决策记录表

步骤5: 更新日志
├── 在"文档更新日志"中记录本次变更
├── 注明变更类型："已解决并归档"
└── 记录变更者和日期

步骤6: 通知相关方
├── 通知所有Agent该瓶颈已解决
├── 提供关联文档链接
└── 说明对下游工作的影响
```
