## 使用说明

### 填写指南
1. **需求ID**：按AUDIO-001、AUDIO-002顺序编号
2. **情绪要求**：清晰描述希望玩家感受到的情绪
3. **技术规格**：明确Godot兼容的格式要求
4. **参考音频**：尽可能提供具体音频链接或详细描述

### 提交流程
1. 核心团队填写需求文档
2. 与音乐同学讨论确认需求细节
3. 音乐同学确认时间安排
4. 开发期间保持沟通，试听初步版本
5. 完成验收，集成到游戏中

### 沟通建议
- 提供具体参考音频，避免抽象描述
- 及时反馈试听感受，具体说明修改方向
- 尊重音乐创作的专业性，给予创作空间
- 验收时关注功能实现而非主观偏好

## 常见音频需求示例

### 背景音乐类
1. **主菜单音乐**（AUDIO-001）
   - 时长：60-90秒循环
   - 情绪：轻松、欢迎感、带有一点冒险色彩
   - 风格：芯片音乐，旋律明快
   - 参考：《元气骑士》主菜单音乐

2. **战斗关卡音乐**（AUDIO-002）
   - 时长：120秒循环
   - 情绪：紧张、节奏感强、有推进感
   - 风格：芯片音乐，鼓点明显
   - 动态变化：随玩家血量降低增加紧张层

### 音效类
1. **攻击音效**（AUDIO-101）
   - 时长：0.3-0.5秒
   - 触发：每次攻击命中
   - 要求：清脆、有打击感、不刺耳
   - 变体：不同武器需要不同音效

2. **拾取音效**（AUDIO-102）
   - 时长：0.5-1秒
   - 触发：拾取金币/物品
   - 要求：悦耳、有满足感、短促
   - 变体：不同物品类型有细微差别

3. **UI音效**（AUDIO-103）
   - 时长：0.1-0.3秒
   - 触发：按钮点击、菜单切换
   - 要求：简洁、不干扰、有反馈感
