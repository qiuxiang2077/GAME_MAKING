# GAME_MAKING - 像素肉鸽动作游戏项目

## 🎮 项目概述

**游戏类型**：俯视角像素风格肉鸽动作游戏  
**参考游戏**：《元气骑士》《喵喵启示录》《饥荒》  
**核心循环**：局内失败进度清零 + 局外成长永久保留  
**美术风格**：像素风格（建议32x32像素尺寸）  
**游戏引擎**：Godot 4.x  
**开发状态**：概念设计阶段  
**暂定标题**：《豆豆核心：像素深渊》

### 核心玩法特色
- **房间清版式推进**：随机生成关卡，俯视角动作射击
- **Boss武器夺取**：击败小Boss后可获取其武器或技能（有冷却时间）
- **双成长系统**：局内临时资源（失败清零）+ 局外永久成长
- **武器多样化**：近战、远程、范围、特殊效果，支持升级分支

## 👥 团队结构

### 核心开发团队（2人）
- **角色**：程序/策划
- **职责**：Godot开发、系统设计、关卡策划、项目管理
- **时间投入**：每人每周约15小时，总计30小时/周

### 外部协作同学
- **美术同学**：像素艺术资源制作（Aseprite/Pixelorama）
- **音乐同学**：音频资源制作（FL Studio/LMMS）

## 🗓️ 16周开发规划

### 阶段1：概念设计（第1-2周）
- 确定故事世界观（一页纸梗概）
- 创建美术风格参考板
- 建立开发环境与Git仓库

### 阶段2：预生产（第3-4周）
- 玩家角色精灵与移动系统
- 基础房间瓦片集与摄像机跟随
- 第一个武器（近战剑）与基础敌人（史莱姆）

### 阶段3：核心系统开发（第5-8周）
- 武器系统基类与切换
- 敌人AI状态机（巡逻→追击→攻击）
- 房间生成系统（随机连接3-5个房间）
- UI框架与音效系统

### 阶段4：内容生产（第9-12周）
- 制作5-8种敌人（含1个Boss）
- 制作10-15种武器/技能
- 实现局外成长与商店系统

### 阶段5：测试与优化（第13-15周）
- 平衡性测试与数值调整
- Bug排查与性能优化（目标60FPS稳定）
- 小范围玩家测试收集反馈

### 阶段6：发布准备（第16周）
- 多平台打包（Windows/macOS/Linux）
- 商店素材准备（截图、图标、封面图）
- 最终测试与版本归档

## 🔧 技术实现要点

### 引擎与工具
- **主引擎**：Godot 4.x（2D引擎对像素游戏支持优秀）
- **美术工具**：Aseprite / Pixelorama
- **版本控制**：Git + GitHub
- **项目管理**：Trello看板 + 飞书文档

### 关键实现设置
1. **像素美术**：纹理导入Filter → Nearest，关闭Mipmaps，启用像素对齐
2. **场景结构**：节点化设计，分离UI层、世界层、实体层
3. **武器系统**：基类继承模式，支持热插拔切换
4. **数据持久化**：Resource格式保存永久进度

### 项目目录结构
```
/Assets/Sprites/     # 角色、敌人、武器精灵图
/Assets/Audio/       # 背景音乐、音效
/Scripts/Player/     # 玩家控制、状态、动画
/Scripts/Enemies/    # 敌人AI、行为树
/Scripts/Weapons/    # 武器基类、具体实现
/Scenes/Characters/  # Godot场景文件
/Scenes/Rooms/
/Scenes/UI/
```
