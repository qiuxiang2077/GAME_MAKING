## 使用说明

### 填写指南
1. **需求ID**：按ART-001、ART-002顺序编号
2. **优先级**：根据开发进度和依赖关系确定
3. **期望日期**：考虑美术同学的时间安排，留出合理缓冲
4. **视觉参考**：尽可能提供具体图片链接或详细描述

### 提交流程
1. 核心团队填写需求文档
2. 与美术同学讨论确认需求细节
3. 美术同学确认时间安排
4. 开发期间保持沟通，提供进度反馈
5. 完成验收，集成到游戏中

### 沟通建议
- 使用微信/飞书及时沟通
- 定期（每周）同步进度
- 遇到问题尽早提出，避免返工
- 验收时提供具体修改建议，而非模糊反馈
